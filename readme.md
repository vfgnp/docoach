
文章読解問題アプリ 仕様・設計まとめ（v0.1）
1. 目的
* 小学4・5・6年生向けの文章読解力向上
* 単なる正誤判定ではなく 「どの読解スキルが苦手か」を可視化する
* iPad単体で完結し、オフラインでも利用可能

2. 対象環境

言語    Swift
UI    SwiftUI
DB    SQLite（ローカル）
ネット接続    不要（将来拡張あり）

4. 問題データ設計
4.1 基本方針
* 問題は 文章＋設問 で構成
* 設問単位で 2〜4個のタグ を付与
* タグは「教えたい力」ではなく 「間違える理由」 を表す

5. タグ設計（定義値）
5.1 MVP必須タグ（8個）

主題把握
要点抽出
指示語
心情理解
因果関係
推論
抽象理解
具体⇄抽象

--------

6.4 QuestionTag（多対多）

QuestionTag
------------
question_id     TEXT

6.5 AnswerLog（学習ログ）

AnswerLog
---------
id              TEXT PRIMARY KEY
question_id     TEXT
is_correct      BOOLEAN
time_sec        INTEGER
answered_at     DATETIME
設計方針
* 上書き禁止
* 1解答 = 1レコード
* 成長・ブレ・定着を全て記録

7. 学習ログ取得指標
指標    用途
正誤    理解度
解答時間    読解負荷
タグ    苦手分類
日時    成長推移
8. 苦手分析ロジック
8.1 タグ別正答率

誤答率 = 誤答数 / 全回答数

8.2 タグ別平均時間

平均解答時間 = time_sec の平均

8.3 苦手度スコア（例）

苦手度 = (誤答率 × 0.7) + (時間超過率 × 0.3)

8.4 出題制御ルール（初期）

苦手タグ問題：60%
通常問題　　：40%
